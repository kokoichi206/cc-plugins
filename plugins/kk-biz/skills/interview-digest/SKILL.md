# 面談メモ整理スキル

`$ARGUMENTS` で指定された面談メモを人ごとに整理し、上の人 (経営層) に相談するための叩き台を 1 ファイルに出力する。

## 引数

- `$ARGUMENTS`: 面談メモのディレクトリまたはファイル。複数指定可 (例: `採用/ ses/面談/`)。未指定の場合は対象を質問する。

## 手順

### 1. メモの収集

- `$ARGUMENTS` がディレクトリなら Glob でメモを列挙する (例: `採用/**/*.md`, `ses/面談/**/*.md`)。
- メモが Markdown 以外 (txt 等) も含む場合は拡張子を広げて拾う。
- 候補者ごとにファイルが分かれていない場合は、Grep で氏名/見出しを手掛かりに人単位へ切り分ける。
- 対象が 0 件なら、探索したパスを示して候補者メモの置き場所を質問する (勝手に対象を広げない)。

### 2. 人ごとにセクション化

候補者 1 人につき 1 セクションを作る。各セクションは以下の構成にする。

- **印象**: 面談で受けた印象を箇条書き (短く、具体的に)。
- **評価軸**: 次の 4 軸を必ず埋める。判断材料が無い軸は「メモに記載なし」と書く。
  - 技術力
  - コミュニケーション
  - 単価 (提示/希望。記載が無ければ「未確認」)
  - 稼働可能日
- **採否・アサイン案**: 採用/見送り/保留、および想定アサイン (どの案件・役割か) の案。
- **最短稼働開始日**: メモから読み取れる開始可能日。不明なら「要確認」。

事実と推測の分離を徹底する。メモに書かれた事実はそのまま、こちらの解釈や推測は `(推測)` を明記して書く。

### 3. 全体サマリと論点

- 全候補者を一覧化した比較表 (氏名 / 評価軸の要約 / 採否案 / 最短稼働日) を先頭付近に置く。
- 「上の人に相談したい論点」を箇条書きでまとめる。例: 単価交渉の余地、複数候補が競合する枠、稼働開始日と案件スケジュールのギャップ、判断に必要な追加情報。

### 4. ファイル出力

- 出力先を決めて Write する (例: 入力ディレクトリ配下の `面談まとめ.md`、または明示された出力先)。出力先が曖昧なら確認する。
- 氏名・単価などの機密はこのファイル内にのみ書き、チャット応答には垂れ流さない。完了報告は出力パスと候補者件数程度に留める。

## 注意事項

- CLAUDE.md のコーディング/対話規約に従う
- メモに書かれた事実と、こちらの推測を明確に分離する (推測には `(推測)` を付す)
- 氏名・単価などの機密は出力ファイル内のみに留め、チャットに出さない
- 全角括弧は使わず半角記号を優先する
- 不明な前提 (単価・稼働日など) を勝手に埋めず、未確認として明示するか質問する
- 探索には fd/rg 相当 (Glob/Grep) を使い、作業スコープを越えて対象を広げない
