# marp スライド生成スキル

`$ARGUMENTS` で指定された元ネタから、人に共有するための marp スライド (Markdown) を生成する。聴衆に合わせて内部用語を翻訳する。

## 引数

- `$ARGUMENTS`: 元ネタとなる docs/ファイル/ディレクトリのパスと、任意のオプション
  - `--audience 顧客`: 顧客など社外向け。専門用語を避け、価値・効果を中心に構成する
  - `--audience 社内`: 社内・チーム向け。技術的な詳細を残してよい
  - 未指定: 聴衆を質問して確認する

## 手順

### 1. 元ネタを収集する

- 指定されたパスを Read で読み込む。ディレクトリの場合は Glob/Grep で関連ファイルを特定して読む
- 元ネタが会話やメモの場合はそのまま要点を抽出する

### 2. 聴衆と目的を確認する

- `--audience` が未指定なら、誰に向けて何を伝えたいスライドかを質問する
- 推測で進めず、不明点は仮定として明示するか質問する

### 3. 構成を設計する

以下を基本構成として、元ネタに応じて取捨選択する:

- タイトル (件名・話者・日付など最小限)
- 課題 / 背景
- 提案 / 解決策
- 詳細 (必要な分だけ)
- まとめ / ネクストアクション

### 4. 内部用語を聴衆向けに翻訳する

- 社内固有の略語・システム名・技術用語を、聴衆が理解できる言葉に置き換える
- 顧客向けの場合は「何ができるか・どんな価値があるか」に変換する
- 翻訳に迷う固有名詞は、元の語を残しつつ補足を添えるか質問する

### 5. marp 記法で出力する

- front-matter に `marp: true` を含める
- スライドは `---` で区切る
- 1 スライド 1 メッセージを守り、情報を詰め込みすぎない

最小例:

```markdown
---
marp: true
theme: default
paginate: true
---

# タイトル

サブタイトル

---

## 課題

- 要点を 1 つに絞る
```

### 6. ビルド手順を案内する

出力後、ローカルでのビルド方法を案内する:

```bash
# HTML として出力
npx @marp-team/marp-cli slides.md -o slides.html

# PDF として出力
npx @marp-team/marp-cli slides.md --pdf
```

## 注意事項

- CLAUDE.md のコーディング/対話規約に従う
- 1 スライド 1 メッセージ。情報を詰め込みすぎない
- 全角括弧を使わず半角記号を優先する
- 内部用語は聴衆に伝わる言葉へ翻訳し、翻訳できない箇所は仮定として明示する
