# 実装の動作確認

実装した機能の動作確認を行う。ブラウザ操作ツール (Playwright MCP, Claude in Chrome 等) を使用してスクリーンショットを撮影し、変更が正しく反映されているか確認する。

## 引数

- `$ARGUMENTS`: 確認対象の説明 (例: "ログインボタンの修正", "ダッシュボード画面の追加")

## 手順

### 1. 出力ディレクトリの作成

- `output/` 配下に、確認対象を示すディレクトリを作成する
- 例: `output/fix-login-button/`, `output/dashboard-feature/`
- ディレクトリ名は変更内容を端的に表すものにする

### 2. 開発サーバーの起動確認

- 開発サーバーが起動しているか確認する
- 起動していない場合はバックグラウンドで起動する

### 3. スクリーンショットの撮影

- 確認が必要なページにアクセスする
- 必要に応じてスクロールやクリックなどの操作を行う
- スクリーンショットを撮影し、作成したディレクトリに保存する
- ファイル名は `01-page-name.png`, `02-after-click.png` のように連番をつける

### 4. 確認結果のレポート

- スクリーンショットの内容を確認し、期待通りの表示になっているか報告する
- 問題があれば指摘する

## 注意事項

- スクリーンショットは `output/` 配下に保存される (git 管理外)
- 開発サーバーが起動していない場合は自動的に起動を試みる
- ブラウザ操作ツールが利用可能な環境で実行すること
