# リファクタリング提案スキル

`$ARGUMENTS` で指定されたコードを分析し、以下の観点からリファクタリングを提案する。

## 分析観点

1. **可読性**: 変数名、関数名、コメントの改善
2. **保守性**: コードの分割、責務の分離
3. **パフォーマンス**: 明らかな非効率の指摘
4. **ベストプラクティス**: 言語固有のイディオムへの準拠

## 出力形式

各提案について以下を明記してください:

- **対象箇所**: ファイル名と行番号
- **現状の問題**: なぜ改善が必要か
- **提案内容**: 具体的な改善案
- **優先度**: HIGH / MEDIUM / LOW

## 注意事項

- 実際のコード変更は行わず、提案のみを行う
- 動作に影響を与えない範囲での改善を提案
- テストコードの変更は提案しない
