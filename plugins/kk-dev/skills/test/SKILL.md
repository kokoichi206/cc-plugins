# テスト生成スキル

`$ARGUMENTS` で指定されたコードに対して適切なテストケースを生成する。

## 手順

1. 対象コードを読み込み、機能を理解する
2. 既存のテストファイル・テストパターンを確認する
3. プロジェクトのテストフレームワークに合わせたテストを生成する

## テスト設計方針

### カバーすべきケース

- **正常系**: 期待される入力と出力
- **境界値**: 最小値、最大値、空、null など
- **異常系**: エラーが発生すべきケース

### テストの書き方

- テスト名は日本語で「何をテストしているか」を明確に
- Arrange-Act-Assert パターンを使用
- 1つのテストで1つの振る舞いを検証

## 出力

- テストファイルを作成または既存ファイルに追加
- テスト実行コマンドを提示
